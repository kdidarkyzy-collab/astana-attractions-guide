# PRD: «Открой Астану»

## 1. Обзор продукта

**Название:** «Открой Астану»  
**Тип:** адаптивный туристический веб-сайт / интерактивный гид  
**Язык первой версии:** русский (с архитектурой для добавления казахского и английского)

«Открой Астану» — визуально выразительный сайт, который помогает жителям и гостям города находить, изучать и удобно планировать посещение достопримечательностей Астаны. Сайт объединяет вдохновляющую подачу фотографий, понятную карту, практическую информацию и тематические маршруты.

## 2. Цель

Создать современный сайт-витрину достопримечательностей Астаны, который:

- вызывает желание исследовать город;
- быстро показывает интересные места рядом или по выбранной теме;
- помогает составить прогулочный маршрут;
- одинаково удобно работает на телефоне, планшете и компьютере.

## 3. Целевая аудитория

| Сегмент | Потребность |
| --- | --- |
| Туристы | Понять, что посмотреть за 1–3 дня, как добраться и сколько времени заложить. |
| Жители Астаны | Открывать новые места, события и идеи для прогулок. |
| Семьи | Находить безопасные и интересные локации для отдыха с детьми. |
| Гости по работе | Быстро выбрать знаковые места поблизости и маршрут на свободный вечер. |

## 4. Основные сценарии

1. Пользователь открывает главную страницу, видит яркую панораму города и подборку главных мест.
2. Пользователь выбирает категорию: архитектура, музеи, парки, семейный отдых, культура, гастрономия.
3. Пользователь открывает карточку достопримечательности, смотрит фото, описание, адрес, время работы, стоимость и расположение на карте.
4. Пользователь сохраняет места в избранное и формирует собственный маршрут.
5. Пользователь выбирает готовый маршрут, например «Астана за один день» или «Вечерняя прогулка по левому берегу».

## 5. Структура сайта

### 5.1. Главная страница

- Полноэкранный первый экран с атмосферным фото/видео Астаны.
- Заголовок: «Открой Астану по-новому».
- Поиск по местам и категориям.
- Быстрые категории с иконками.
- Блок «Главные достопримечательности».
- Интерактивная мини-карта.
- Блок готовых маршрутов.
- Подборка «Сейчас в городе» — сезонные рекомендации или события.
- Фото-галерея / вдохновляющая лента.
- Подвал с контактами, соцсетями, языками и ссылками.

### 5.2. Каталог достопримечательностей

- Сетка карточек с большими фотографиями.
- Фильтры: категория, район/берег, время посещения, стоимость, подходит для детей, доступность.
- Сортировка: популярные, поблизости, новые, бесплатные.
- Поиск по названию и ключевым словам.
- Отображение результатов списком и на карте.

### 5.3. Страница достопримечательности

Обязательные блоки:

- название, категория и короткое описание;
- крупная фотогалерея;
- история и интересные факты;
- адрес и точка на карте;
- часы работы, стоимость, сайт/контакты (при наличии);
- ориентировочная длительность посещения;
- теги: «с детьми», «бесплатно», «в помещении», «доступно для маломобильных гостей»;
- кнопки «Добавить в маршрут», «В избранное», «Поделиться»;
- ближайшие интересные места;
- блок «Как добраться».

### 5.4. Карта

- Интерактивная карта с маркерами всех объектов.
- Кластеризация маркеров при уменьшении масштаба.
- Фильтрация без перезагрузки страницы.
- Карточка места при клике на маркер.
- Определение текущего местоположения — только после согласия пользователя.
- Построение маршрута через внешний картографический сервис.

### 5.5. Маршруты

Готовые тематические маршруты:

- «Астана за 1 день»;
- «Современная архитектура»;
- «Левый берег вечером»;
- «Семейный выходной»;
- «Музеи и история»;
- «Бесплатная прогулка по городу».

Для каждого маршрута показывать длительность, расстояние, количество точек, порядок посещения и карту.

### 5.6. Избранное и личный маршрут

- Сохранение мест без обязательной регистрации (локально в браузере).
- Возможность удалить, изменить порядок и поделиться подборкой.
- Расчёт примерной длительности и расстояния.
- Опционально: авторизация для синхронизации между устройствами.

## 6. Контент первой версии

Начальный каталог должен включать не менее 20 достопримечательностей. Примеры ключевых объектов:

- монумент «Байтерек»;
- мечеть Хазрет Султан;
- Дворец мира и согласия;
- Национальный музей Республики Казахстан;
- мечеть Абу Наср аль-Фараби;
- Триумфальная арка «Мәңгілік Ел»;
- набережная реки Есиль;
- Ботанический сад;
- парк влюблённых;
- выставочный центр EXPO и сфера «Нур Алем»;
- театр «Астана Опера»;
- театр «Астана Балет»;
- Президентский парк;
- площадь Независимости;
- Центральный парк;
- ТРЦ Khan Shatyr.

Для каждого объекта нужны: уникальное описание, качественная обложка, 3–8 фотографий, координаты, категория, адрес, практические данные и дата последней проверки информации.

## 7. Требования к дизайну

### Визуальная концепция

Стиль — современный, чистый и премиальный, с акцентом на футуристичную архитектуру и простор Астаны. Визуальный язык должен сочетать холодные небесные оттенки, белый фон, глубокий синий и тёплый золотой акцент.

### Интерфейсные принципы

- Большие качественные фотографии и выразительная типографика.
- Воздушные отступы, скруглённые карточки, мягкие тени.
- Плавные ненавязчивые анимации при прокрутке и наведении.
- Контрастный, доступный текст.
- Читаемая навигация с фиксированной шапкой.
- Светлая тема в первой версии; тёмная тема — желательное развитие.

### Адаптивность

- Mobile-first подход.
- На мобильном: нижняя навигация или компактное меню, удобные крупные кнопки, карта на весь экран.
- На десктопе: широкая сетка карточек, расширенные фильтры, карта рядом со списком.

## 9. Нефункциональные требования

- Загрузка ключевого контента на мобильной сети — до 3 секунд при оптимизированных изображениях.
- Доступность: контраст текста, управление с клавиатуры, alt-тексты, семантическая разметка.
- SEO: уникальные title/description, человекопонятные URL, sitemap.xml, Open Graph-метаданные.
- Фото загружаются в современных форматах WebP/AVIF, с lazy loading.
- Защита форм от спама и базовая защита пользовательских данных.
- Корректная работа в актуальных Chrome, Safari, Firefox и Edge.

## 10. Рекомендуемый технологический стек

- **Frontend:** Next.js или React + TypeScript.
- **Стили:** Tailwind CSS или CSS Modules.
- **Контент и администрирование:** Strapi, Sanity, Directus или собственная админ-панель.
- **База данных:** PostgreSQL.
- **Карты:** 2GIS, Google Maps или Mapbox — выбрать с учётом лицензии, покрытия и локальных данных.
- **Хранение изображений:** CDN-совместимое облачное хранилище.
- **Аналитика:** Яндекс.Метрика и/или Google Analytics с учётом требований к cookies.

## 12. Метрики успеха

- не менее 60% посетителей открывают карточку хотя бы одного места;
- не менее 15% посетителей используют поиск, фильтры или карту;
- не менее 8% посетителей открывают готовый маршрут;
- средняя глубина просмотра — от 3 страниц;
- Core Web Vitals соответствуют рекомендациям Google.

## 13. Этапы реализации

1. **Исследование и контент-план:** список мест, источники данных, фотографии, категории.
2. **UX/UI-дизайн:** прототип, дизайн-система, макеты для desktop и mobile.
3. **MVP-разработка:** главная, каталог, карточка места, карта, адаптивность.
4. **Наполнение:** публикация первых 20+ достопримечательностей и 5+ маршрутов.
5. **Тестирование:** устройства, браузеры, скорость, доступность, SEO.
6. **Запуск и развитие:** аналитика, новые языки, отзывы, календарь событий, персональные рекомендации.

## 14. Критерии готовности MVP

- Главная страница, каталог, карта и не менее 20 карточек мест опубликованы.
- Поиск и фильтры работают корректно.
- У каждой карточки есть фотографии, описание, координаты и практическая информация.
- Сайт адаптирован под мобильные устройства.
- Пользователь может сохранять места в избранное.
- Сайт проходит базовые проверки производительности, SEO и доступности.
